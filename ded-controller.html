A multi-axis motion control system for a Directed Energy Deposition (DED) additive manufacturing machine. MATLAB generates and sends motion commands via UART to microcontrollers, which execute movements and return feedback. X/Y axes use stepper motors with SPI drivers. Z-axis uses a servo. A custom PCB reduced wiring complexity and improved reliability.

// What is DED?
Project Context

Directed Energy Deposition (DED) is an additive manufacturing process that uses a focused laser or electron beam to fuse metallic feedstock — wire or powder — layer by layer as it is deposited.

This project involved building the motion control system for a DED machine: coordinating X, Y, and Z axis movement. MATLAB acts as the command interface, sending motion commands over UART to the microcontrollers.

The microcontrollers parse the commands, control the motors, and send feedback back to MATLAB. A custom PCB ties the hardware together cleanly.

// Hardware Stack
System Components
  • MATLAB — control command generation on host machine
  • Arduino microcontrollers — command parsing and motor execution
  • X/Y axes — stepper motors via SPI motor drivers
  • Z-axis — servo motor with built-in driver
  • UART — bidirectional comms between MATLAB and MCUs
  • SPI — communication with stepper motor drivers
  • Custom PCB — consolidated wiring, improved reliability
// System Architecture
How It Works
01
Command Generation
MATLAB generates motion commands based on the desired tool path. Commands are formatted and sent over UART to the microcontroller.
02
UART Communication
Serial communication between MATLAB and Arduino. Commands sent down, feedback and status sent back up. Reliable framing and parsing handled in firmware.
03
Command Parsing
Microcontroller parses incoming command strings, extracts axis targets and speed parameters, and executes corresponding motor control routines.
04
Motor Execution
X/Y steppers driven via SPI motor drivers. Z-axis servo controlled via PWM. Coordinated execution achieves synchronized multi-axis motion.
// Challenges
What Was Hard
  • Multi-axis synchronization — getting all axes to move together reliably
  • Reliable UART communication — handling framing, timing, and noise
  • Hardware complexity — managing multiple drivers, MCUs, and wiring
  • Custom PCB design to consolidate connections and reduce failure points
  • Integration testing across the full MATLAB → MCU → motor chain
// Results
Outcome
  • Coordinated multi-axis motion achieved across X, Y, and Z
  • Reliable UART communication between MATLAB and microcontrollers
  • Custom PCB significantly improved reliability and wiring cleanliness
  • System successfully integrated into DED machine hardware setup
