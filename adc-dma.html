Multi-channel analog data acquisition using ADC scan mode and DMA circular buffers on an STM32F446RE. Data is streamed over UART in real time for visualization in RealTerm. The goal was to reduce CPU load by letting DMA handle transfers autonomously while the MCU stays free for other work.

// System Overview
What This Does

The system continuously samples multiple analog channels and streams the data in real time — all without tying up the CPU. ADC scan mode handles sequential channel sampling. DMA circular mode handles the transfers autonomously.

Once DMA fills the buffer, the CPU reads the values, converts raw counts to voltage, and sends them over UART. The CPU is free for other tasks during the DMA transfer itself.

This is a foundational pattern in embedded data acquisition — understanding it well unlocks a lot of more complex firmware work downstream.

// Hardware Setup
System Spec
  • STM32F446RE — ARM Cortex-M4, 180MHz
  • ADC in scan mode — multiple channels, single trigger
  • DMA in circular mode — peripheral-to-memory transfer
  • Continuous conversion mode enabled
  • 12-bit resolution — 0 to 4095 raw, 0 to 3.3V converted
  • UART transmission to host PC
  • RealTerm used for real-time data display
  • ST-LINK debugger for bring-up and verification
// Firmware Architecture
How It's Implemented
ADC Configuration
  • Scan mode enabled for multi-channel sequential sampling
  • Continuous conversion mode — no manual retrigger needed
  • DMA request enabled on ADC peripheral
  • Each channel assigned a rank in the sequence
  • Sample time configured per channel sensitivity
DMA Configuration
  • Direction: peripheral to memory (ADC DR → buffer)
  • Circular mode — buffer wraps automatically
  • Half-word transfer size to match 12-bit ADC resolution
  • DMA complete callback used to trigger UART transmission
  • No CPU involvement during the transfer itself
// Debugging
Problems Encountered
Non-Continuous Data Transfer
DMA was stopping after one complete buffer cycle. Root cause: circular mode not correctly enabled in DMA configuration. Fixed by correctly setting the circular bit in DMA stream config.
Peripheral Configuration Mismatch
ADC scan mode and DMA were enabled independently but not linked correctly. The ADC DMA request enable bit was missing, so DMA was never triggered. Identified by stepping through registers in the debugger.
Output Verification
Used ST-LINK live watch to confirm buffer was filling correctly before wiring up UART. Then verified voltage values against a known reference signal on the input.
// Results
What Worked
  • Continuous real-time sampling across multiple channels
  • CPU load significantly reduced — DMA handles all transfers
  • Stable, reliable data streaming over UART
  • Accurate voltage conversion from raw ADC counts
  • System runs indefinitely without manual intervention
