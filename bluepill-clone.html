// Design Features
What Was Built Into This Board
Original Blue Pill Issues
  • Wrong USB D+ pull-up value (enumeration failures)
  • Minimal decoupling capacitors on VDD pins
  • No solid ground plane — poor return paths
  • Crystal routing not following datasheet guidance
  • Crowded, hard-to-rework layout
  • No BOOT0 switch — requires jumper wires
This Design
  • AMS1117 regulator — clean USB 5V to 3.3V conversion
  • Decoupling capacitors placed directly at MCU VDD pins
  • Bulk capacitor for power supply stability
  • Bottom layer used as continuous ground plane
  • External 8MHz oscillator for stable clock source
  • SPDT switch for BOOT0 — no jumpers needed
  • ST-LINK, UART, and I2C pins clearly exposed
// Process
How I Built It

Started by thoroughly reading the STM32F103C8T6 datasheet — power supply filtering, decoupling requirements, reset circuit, oscillator circuit, and pin mapping.

Used the Blue Pill as a reference form factor but redesigned the schematic from scratch, correcting every known issue — pull-up values, decoupling placement, and ground topology.

PCB layout followed a deliberate order: ground plane first, then power traces, then the oscillator circuit, then digital I/O. Decoupling caps placed as close as physically possible to each VDD pin.

Outcome: a manufacturable two-layer development board with real-world PCB design practices applied throughout.

// Key Sub-Circuits
Schematic Blocks
AMS1117 Voltage Regulation
USB 5V stepped down to 3.3V using AMS1117. Input and output bulk caps for stability. Output decoupled close to MCU VDD pins.
External 8MHz Oscillator
Crystal with matched load caps per datasheet. Traces kept short and routed away from digital switching signals for clock stability.
BOOT0 SPDT Switch
Physical switch to toggle BOOT0 between GND (run from flash) and 3.3V (enter bootloader) — eliminates the need for jumper wires during development.
